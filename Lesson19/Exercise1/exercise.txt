<h3>Lesson 19, Exercise 1</h3>
<p>Let&rsquo;s take a look at a wireframe mockup of a page, where we have an image block, summary block, and text block. The image and summary blocks share a row, where the image take up 60% of the width and the summary occupies the remaining space via the flex-grow property. Below those blocks sits the the text, which occupies&nbsp;<code>100%</code>&nbsp;of the width.</p>
<p>It looks fine on a large monitor. But if we take the browser window and scale it down to a more mobile-size width, things start looking a bit funky. The summary wraps to its own line, which is what we want, but the image still only occupies&nbsp;<code>60%</code>&nbsp;of the width, which isn&rsquo;t what we want. To fix the layout for mobile, let&rsquo;s use a media query to change the image width to&nbsp;<code>100%</code>&nbsp;when the browser is less than&nbsp;<code>600px</code>&nbsp;wide.</p>
<p><strong>Tasks</strong></p>
<ul>
<li>Create a media query within the&nbsp;<code>.article__image</code>&nbsp;selector via the&nbsp;<code>@media</code>&nbsp;keyword.</li>
<li>Set the query to be the&nbsp;<code>max-width</code>&nbsp;property with a value of&nbsp;<code>599px</code></li>
<li>After the query list place a set of curly brackets:&nbsp;<code>{ }</code></li>
<li>Inside of the curly brackets, place the properties that we want applied when the width of the browser window is&nbsp;<code>599px</code>&nbsp;or less, which is to set the&nbsp;<code>width</code>&nbsp;to&nbsp;<code>100%</code></li>
<li>Review the rendered page. Resize it and ensure that the image changes relative width when the browser is less than&nbsp;<code>600px</code>&nbsp;wide</li>
<li>Check the compiled CSS. Outside of the&nbsp;<code>.article__image</code>&nbsp;selector there should be a media query for&nbsp;<code>.article__selector</code>&nbsp;with a&nbsp;<code>max-width</code>&nbsp;of&nbsp;<code>599px</code>&nbsp;and the&nbsp;<code>width</code>&nbsp;property set to&nbsp;<code>100%</code></li>
</ul>
